## Трекер задач сотрудников

### ## Employee CRUD API (Django REST Framework)

В проекте реализован REST API для управления сотрудниками с использованием Django REST Framework.

### Реализованный функционал
- Получение списка сотрудников
- Получение сотрудника по id
- Создание сотрудника
- Обновление сотрудника (частичное и полное)
- Удаление сотрудника

API реализован на базе `ModelViewSet` и `DefaultRouter`.

### Эндпоинты

| Метод | URL | Описание |
|------|-----|----------|
| GET | /api/employees/ | Список сотрудников |
| POST | /api/employees/ | Создание сотрудника |
| GET | /api/employees/{id}/ | Получение сотрудника |
| PATCH | /api/employees/{id}/ | Частичное обновление |
| PUT | /api/employees/{id}/ | Полное обновление |
| DELETE | /api/employees/{id}/ | Удаление сотрудника |

### Пример POST-запроса
```json
{
  "full_name": "Иванов Иван Иванович",
  "position": "Менеджер",
  "email": "ivanov@example.com",
  "is_active": true
}
```
### Валидации

- full_name - минимальная длина 5 символов (сериализатор)
- для активного сотрудника рекомендуется указывать email
- валидации вынесены в модели (через clean() и full_clean())

## + 
В API добавлен минимальный поиск по ФИО сотрудников с использованием DRF SearchFilter. Поиск выполняется через query-параметр search и поддерживает частичное совпадение.

`GET /api/employees/?search=Иванов`
